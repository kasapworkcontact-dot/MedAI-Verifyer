1. บทสรุปโครงการ (Executive Summary)

โครงการนี้เป็นการสานต่อและปรับปรุงระบบจากโปรเจกต์เดิม เพื่อให้สามารถทำงานได้จริงและมีความสมบูรณ์พร้อมสำหรับการนำเสนอ โดยจะมุ่งเน้นไปที่การสร้าง Web Application เพื่อเป็นส่วนติดต่อผู้ใช้งาน (User Interface) และการเปลี่ยนระบบอ่านข้อความ (OCR) จากเดิมมาใช้ Large Language Model (LLM - Gemini API) เพื่อเพิ่มความแม่นยำในการอ่านลายมือภาษาไทยและภาษาอังกฤษบนแบบฟอร์ม นอกจากนี้ จะมีการพัฒนาระบบ Human-in-the-Loop เพื่อให้ผู้ใช้งานสามารถตรวจสอบและแก้ไขความถูกต้องของข้อมูลที่ AI ตรวจจับได้ ทั้งในส่วนของจำนวนอุปกรณ์ และรูปภาพอุปกรณ์ที่โมเดลมีความมั่นใจต่ำ (Low Confidence)

2. ขอบเขตการทำงาน (Scope of Work)

งานพัฒนาจะถูกแบ่งออกเป็น 3 ส่วนหลัก (Modules) ดังนี้:

2.1 Web Application (Frontend & Backend POC)

พัฒนา Web App สำหรับใช้งานเป็น Proof of Concept (POC) โดยมีฟีเจอร์หลักดังนี้:

Responsive Design (Mobile-Friendly): ออกแบบส่วนติดต่อผู้ใช้งาน (UI) ให้รองรับการแสดงผลและใช้งานได้ดีบนทุกหน้าจอ โดยเฉพาะบนสมาร์ทโฟนและแท็บเล็ต เพื่อให้เจ้าหน้าที่สามารถถ่ายรูปและอัปโหลดข้อมูลหน้างานได้อย่างสะดวก

Image & Form Upload: แบ่งหน้าจอการอัปโหลดออกเป็นสัดส่วน ได้แก่

อัปโหลดภาพอุปกรณ์ก่อนผ่าตัด (Pre-operation)

อัปโหลดภาพอุปกรณ์หลังผ่าตัด (Post-operation)

อัปโหลดภาพแบบฟอร์มบันทึกการผ่าตัด (Checklist Form)

Low-Confidence Annotation Feature: ระบบจะดึงภาพที่ถูก Crop จากโฟลเดอร์ low_confidence_crops (ภาพที่โมเดล YOLO มีความมั่นใจ < 80%) มาแสดงผลบน Web App เพื่อให้ผู้ใช้งาน (Human) เป็นคนตรวจสอบและระบุ (Label) ว่าภาพนั้นคืออุปกรณ์ชนิดใดด้วยตนเอง

Human Verification Dashboard: หน้าจอสรุปผลเปรียบเทียบระหว่าง "จำนวนที่ YOLO นับได้รวมกับที่คน Label" กับ "จำนวนที่ OCR อ่านได้จากแบบฟอร์ม" หากข้อมูลไม่ตรงกัน ผู้ใช้งานสามารถกดแก้ไขข้อมูลให้ถูกต้องได้ก่อนบันทึกผลลัพธ์สุดท้าย

2.2 OCR Module Refactoring (Gemini API Integration)

ถอดระบบ OCR เดิม (Typhoon + Tesseract) ที่มีปัญหาความแม่นยำออก

เขียนสคริปต์เชื่อมต่อกับ Google Gemini API (Plan Free) เพื่อทำหน้าที่เป็น OCR Engine

ทำ Prompt Engineering เพื่อให้ Gemini สกัดข้อมูลจากภาพแบบฟอร์ม (ลายมือไทย/อังกฤษ) ออกมาเป็นโครงสร้างข้อมูลตาราง (JSON) ที่พร้อมนำไปเปรียบเทียบกับผลลัพธ์จาก YOLO

2.3 System Integration

นำโค้ด Python สกัดภาพ (crop-feature.txt) และโมเดล YOLO เดิมของลูกค้า (.engine / .onnx) มาเชื่อมต่อเข้ากับ Backend ของ Web App ให้ทำงานร่วมกันได้อย่างสมบูรณ์แบบ (End-to-End Flow)

หมายเหตุ: ขอบเขตงานนี้เป็นการทำเพื่อส่งโปรเจกต์จบ (POC) ระบบจะไม่ได้ถูกปรับแต่งเพื่อรองรับผู้ใช้งานพร้อมกันจำนวนมาก (Scale) และใช้ Gemini API แบบฟรี ซึ่งอาจมีข้อจำกัดเรื่อง Rate Limit เบื้องต้น
